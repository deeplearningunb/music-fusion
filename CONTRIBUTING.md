# Contributing

When contributing to this repository, please first discuss the change you wish to make via issue with the owners of this repository before making a change. 

Please note we have a code of conduct, please follow it in all your interactions with the project.

## Our Development Process
We use GitHub to sync code to and from our internal repository. We'll use GitHub
to track issues and feature requests, as well as accept pull requests.

We are one branch to intern development `dev`, but the `master` branch is most stable branch. Therefore, it should be used as a basis for external contribution.

### `branches` (ONLY INTERN PROCESS)

The `master` branch represent the most stable branch, it is a production branch. The `dev` branch represent the code in development, before released. It will be necessary to create auxiliary branches for the development of each a new feature or fix bug, these will be branching `dev`. 

## Pull Requests
We actively welcome your pull requests.

1. Fork the repo and create your branch from `master`.
2. If you've changed APIs, update the documentation.

## Issues
We use GitHub issues to track public bugs. Please ensure your description is
clear and has sufficient instructions to be able to reproduce the issue.

### Tags
The tags used in this project are:
- `bug` - The bug is identified in the project
- `enhancement` - New feature is created in the project.
- `refactoring` - Refactoring is required in source code.
- `proposal` - Improvement proposal or new feature is made.
- `documentation` - The explanation of the project or code must be documented.
